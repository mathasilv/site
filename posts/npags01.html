POR DENTRO DO PROJETO: NPA GROUND STATION

DATA: 2025-12-27

Hoje quero apresentar em detalhes a NPA Ground Station, uma solução de software completa que desenvolvi para recepção, decodificação e visualização de telemetria via rádio (LoRa/SDR) e rede.

O objetivo principal deste projeto foi resolver um problema comum em estações terrestres: a rigidez. Normalmente, para adicionar um novo sensor ou satélite, é preciso reescrever o código-fonte. Na NPA-GS v5.2, isso mudou radicalmente.

1. O Coração: Decoder Engine Dinâmico

A maior inovação do projeto está no núcleo de processamento. Criei um Decoder Engine que elimina a necessidade de programar "parsers" manualmente.

Todo o protocolo de comunicação é definido em arquivos YAML. O motor lê esse arquivo e aprende dinamicamente como extrair bits, bytes, inteiros, floats e até arrays de dentro dos pacotes brutos. Isso permite que a estação suporte múltiplos satélites ou sensores simultaneamente, usando o sistema de Multi-Decoder para detecção automática.

2. Hardware e SDR (Software Defined Radio)

Para a camada física, integrei o GNU Radio diretamente na aplicação Python. Isso permite conectar dongles RTL-SDR baratos e receber sinais reais do ar.

  • Waterfall em Tempo Real: Implementei um "sniffer" que captura dados da FFT e desenha o espectro de radiofrequência na interface, permitindo visualizar o sinal chegando.
  • Modo Híbrido: Se você não tem um rádio por perto, o sistema possui um receptor UDP embutido, permitindo testar a estação via rede ou receber dados de outros computadores.

3. Interface Gráfica e Ferramentas

A interface foi construída com CustomTkinter para um visual moderno e responsivo. Além de exibir os dados decodificados e o espectro, desenvolvi um Editor Integrado. Você pode criar, editar e validar os schemas YAML dos seus sensores sem nunca fechar a aplicação.

Todos os dados recebidos são automaticamente rotacionados e salvos em formato JSONL (JSON Lines), garantindo que nenhum pacote de telemetria seja perdido, mesmo em operações de longa duração.

Resumo das Tecnologias

  • Linguagem: Python 3.10+
  • GUI: CustomTkinter & Matplotlib
  • DSP/Rádio: GNU Radio & Osmocom SDR
  • Dados: YAML (Configuração) & JSONL (Logs)

O projeto é Open Source e convido todos a contribuírem ou testarem com seus próprios sensores LoRa.
